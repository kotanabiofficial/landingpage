# UI/UX Audit Checklist

Use this checklist to perform a comprehensive audit of a digital experience.

## 1. Usability & Functionality
- [ ] **Core Task Completion**: Can the user complete the primary goal (e.g., purchasing or signing up) without confusion?
- [ ] **Navigation**: Is the menu structure intuitive? Is the user's current location clear?
- [ ] **Form UX**: Are forms as short as possible? Are labels clear? Is error validation helpful?
- [ ] **Search**: If present, does search return relevant results?

## 2. Visual Design & Brand
- [ ] **Consistency**: Are colors, fonts, and spacing consistent throughout the experience?
- [ ] **Hierarchy**: Is it immediately clear what the most important element on the page is?
- [ ] **Visual Clarity**: Is there enough contrast for readability?
- [ ] **Negative Space**: Does the layout feel cluttered or "breathable"?

## 3. Interaction & Feedback
- [ ] **Clickable Elements**: Do all buttons and links have hover/active states?
- [ ] **Wait States**: Is there visual feedback for loading or processing?
- [ ] **Success/Error States**: Are success and error messages clear and actionable?
- [ ] **Transitions**: Are animations smooth and functional (not distracting)?

## 4. Mobile & Responsive
- [ ] **Touch Targets**: Are buttons at least 44x44px for easy tapping?
- [ ] **Thumb Zone**: Are primary actions within reach of a user's thumb?
- [ ] **Responsive Breakpoints**: Does the layout break or overlap at common screen sizes?
- [ ] **Mobile Speed**: Does the page feel snappy on a mobile connection?

## 5. Accessibility (A11y)
- [ ] **Contrast**: Do text/background color ratios meet WCAG AA standards (4.5:1)?
- [ ] **Screen Readers**: Is semantic HTML used correctly? Do images have alt text?
- [ ] **Focus States**: Is the keyboard focus ring visible and logical?

## Audit Workflow
1. **Identify Persona**: Who is the user and what is their goal?
2. **Cognitive Walkthrough**: Walk through the core user journey.
3. **Heuristic Evaluation**: Apply Nielsen's 10 heuristics.
4. **Identify Friction**: Locate points where users drop off or get confused.
5. **Prioritize Fixes**: Label findings as High (Critical), Medium (Important), or Low (Polishing).
